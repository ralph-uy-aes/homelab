# Lecture 1 - Software Security

#### Software Security
- Secure systems can function in spite of attacks
- Aims to bring security in
- Three pillars: Risk management, Touchpoints, Knowledge
- Ex. A bug used to be default credentials for all products (mass-produced) like routers
    - Can be used to connect to private networks using default credentials

#### Common Knowledge
- Mapping of software security knowledge catalogs to various software artifacts are software security best practices
- CVE, CWE, CAPEC are all publicly released in websites (NVD or National Vulnerability Database for example)
