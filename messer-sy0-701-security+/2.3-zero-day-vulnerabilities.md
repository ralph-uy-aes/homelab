# 2.3 Zero Day Vulnerabilities

#### Vulnerabilities
- Many applications have vulnerabilities
    - Just haven't been found yet
- People are working hard to find the next big vulnerability
    - Researchers want to share vulnerabilities to developers
    - Attackers want to keep the vulnerabilities for personal gain

#### Zero-day Attacks
- Searching for unknown vulnerabilities
- Exploits are created for these vulnerabilities
- Vendor has no idea the vulnerability exists
    - No patch yet
- It is a race to exploit the vulnerability or create a patch
- Difficult to defend against the unknown

#### Zero-day Attacks in the Wild
- April 2023 - Chrome zero-day
    - Memory corruption, sandbox escape
- May 2023 - Microsoft zero-day patch
    - Secure boot zero-day vulnerability
    - Attacker can run UEFI-level self-signed code
- May 2023 - Apple iOS and iPadOS zero-days
    - 3 zero-day patches
    - Sandbox escape, disclosure of sensitive information, arbitrary code execution
    - Active exploitations
