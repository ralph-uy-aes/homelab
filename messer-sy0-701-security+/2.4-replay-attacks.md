# 2.4 Replay Attacks

#### Replay Attack
- Useful information is transmitted over the network
    - Crafty hacker will take advantage of this
- Need to access raw network data
    - Network tap, ARP poisoning
    - Malware on victim computer
- Gathered information may help the attacker
    - Replay data to appear as someone else
- Not an on-path attack
    - An on-path attack could be used to get information for a replay attack

#### Pass the Hash
- Client sends a username and hashed password to a server
- Attacker gets a copy of these
- Attacker sends their own auth request, using the hash, thereby doing a replay attack
- Encryption or salting the hash can help prevent this
    - Server is configured to not accept the same hash twice in a row

#### Browser Cookies and Session IDs
- Cookies
    - Information stored on your computer by the browser
- Used for tracking, personalization, session management
    - Not executable, not generally a security risk
    - Unless someone gets access to them
- Could be considered a privacy risk
    - Lots of personal data in there
- Session IDs are often stored in the cookie
    - Maintains sessions across multiple browser sessions

#### Session hijacking or sidejacking
- Victim sends session ID to a server
- Attacker somehow gets the same session ID and sends it to the server as well
- Server gives access to both due to same session ID

#### Header Manipulation
- Info gathering
    - Wireshark, kismet
- Exploits
    - XSS
- Modify headers
    - Tamper, Firesheep, Scapy
- Modify cookies
    - Cookies manager, or firefox addons

#### Prevent Session Hijacking
- Encrypt end to end
    - Can't capture session ID if its encrypted
    - Additional load on web server (HTTPS)
- Encrypt end to somewhere
    - At least capture over a local wireless network
    - Still in the clear for most of the journey
    - Personal VPN
