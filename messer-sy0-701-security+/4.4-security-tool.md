# 4.4 Security Tools

#### Security Content Automation Protocol (SCAP)
- Many different tools = many different sources of logs/information
    - All of them could identify the same vulnerability, but format differently
- SCAP standardizes all formats for all devices
    - Tools identify and act on the same criteria
    - Created by NIST

#### Using SCAP
- SCAP content can be shared between tools
    - Focused on configuration compliance
    - EAsily detect applications with known vulns
- Useful in large environments
    - Many different OS and apps
- Specification standard enables automation
- Automation Types
    - Ongoing monitoring
    - Notification and alerting
    - Remediation of noncompliant systems

#### Benchmarks
- Apply security best-practices to everything
    - OS, Cloud, Mobile devices
    - Bare minimum for security settings
- Extensive library for popular security benchmarks found in cisecurity.org/cis-benchmarks

#### Agents/agentless
- Check to see if the device is in compliance
    - Install a software agent onto the device
    - Run an on-demand agentless check
- Agents can usually provide more detail
    - Always monitoring for notifications
    - Must be updated and maintained
- Agentless runs without a formal install
    - Performs the check, then disappears
    - Does not require ongoing updates
    - Will not inform nor alert if not running

#### SIEM
- Security Information and Event Management
- Collects logs of security alerts in real time
- Aggregates logs and stores long term
- Data correlation between diverse data types
- Forensic analysis is aided by SIEMs

#### Anti-virus
- AKA anti-malware

#### Data Loss Prevention (DLP)
- Stops sensitive data from being transferred over the network
- Prevents transfers of SSN, card numbers, medical records
- Stops the data before the attacker gets it
    - Prevents leaks
- Many sources mean more solutions needed
    - Endpoint clients, cloud, email, collaboration tools

#### SNMP
- Simple Network Management Protocol
    - Database of data (MIB) - Management Information Base
    - Database contains OIDs - Object Identifiers
    - Poll devices over udp/161
- Requests statistics from a device
    - Poll devices at fixed intervals
    - Create a graph of response information

#### SNMP Traps
- Most SNMP operations expect a poll
    - Polling is done in fixed intervals
- SNMP traps can be configured on the monitored device
    - Communicates over udp/162
- Set a threshold for alerts
    - If the number of CRC errors increases by 5, send a trap
    - Monitoring station can react immediately

#### NetFlow
- Gather traffic statistics from all traffic flows
    - Shared communication between devices
- NetFlow
    - Standard collection method
    - Many products and options
- Probe and collector
    - Probe watches network communication
    - Summary records are sent to the collector
- Often paired with a separate reporting app or dashboard

#### Vulnerability Scanners
- Usually minimally invasive
- Port scan
- Identify systems
- Test both externally and internally
- Gather as much as possible
