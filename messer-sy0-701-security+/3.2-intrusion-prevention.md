# 3.2 Intrusion Prevention

#### Intrusion Prevention System (IPS)
- Monitors network traffic, blocks malicious traffic
- Intrusions can be
    - Exploits against OS, apps
    - Buffer overflows, XSS, etc
- Detection vs Prevention
    - Intrusion Detection Systems (IDS) only alert of intrusion
    - Intrusion Prevention Systems (IPS) proactively stop intrusions

#### Failure Modes
- Devices can fail, but in different ways
- Fail-open
    - When a system fails, data continues to flow through system
    - No security processes, but network is available
- Fail-closed
    - When a system fails, data does not flow through system
    - No security processes, and network is not available

#### Device Connections
- Active monitoring
    - System is connected inline, meaning everything has to pass through IPS
    - Data can be blocked in real-time
    - Intrusion prevention is commonly active
    - Usually this is the case because we want real-time security
    - Malicious traffic is immediately identified and dropped
- Passive Monitoring
    - System is connected via switch, does not pass through IPS
    - A copy of traffic is sent to IPS from the switch either through port mirror, switch port analyzer (SPAN) or network tap
    - Cannot be blocked in real time
    - Less secure, but bypasses IPS SPOF or strict IPS rules
    - Common with IDS
