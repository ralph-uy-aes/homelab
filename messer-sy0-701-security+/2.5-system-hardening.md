# 2.5 System Hardening

#### System Hardening
- Many and varied
    - Windows, Linux, iOS, Android, etc
- Updates
    - OS updates, security patches
- User accounts
    - Password rules
    - Account limitations, need to know, least privilege
- Network access and security
    - Limit network access
- Monitor and secure
    - Antivirus
    - Endpoint detection

#### Encryption
- Prevent access to application data files
    - File system encryption
    - Windows EFS
- Full disk encryption (FDE)
    - Encrypt everything on the drive
    - Windows BitLocker, macOS FileVault
- Encrypt all network communication
    - VPN
    - Application encryption (HTTPS, etc)

#### Endpoint
- User's access
    - Apps and data
- Stop attackers
    - Inbound and outbound (outsider and insider threats)
- Many different platforms
    - Mobile, desktop
- Protection is multi-faceted
    - Defense in depth

#### Endpoint Detection and Response (EDR)
- A different method of threat protection
    - Scale to meet the increasing number of threats
- Detect a threat
    - Signatures aren't the only detection tool
    - Behavioral analysis, machine learning, process monitoring
    - Lightweight agent on the endpoint
- Investigate a threat
    - Root cause analysis (RCA)
- Respond to a threat
    - Isolate system, quarantine threat, rollback to config
    - API driven, no user or tech intervention required

#### Host-based firewall
- Software-based
    - Personal firewall, runs on every endpoint
- Allow or disallow incoming or outgoing application traffic
    - Control by application process
    - View all data
- Identify and block unknown processes
    - Stop malware before it can start
- Managed centrally (on each individual system)

#### Finding Intrusions
- Host-based intrusion prevention systems
    - Recognize and block known attacks
    - Secure OS and application configs, validate incoming service requests
    - Often built into endpoint protection software
- HIPS identification
    - Signatures, heuristics, behavioral
    - Buffer overflows, registry updates, writing files into Windows folder
    - Access to non-encrypted data

#### Open ports and services
- Every open port is a possible entry point
    - Close everything but what is required
- Control access with a firewall
    - Next-gen firewall is ideal (NGFW)
- Unused or unknown services
    - Installed with the OS or from other applications
- Use nmap to verify what ports are open on your system

#### Default password changes
- Every network device has a management interface
    - Which come with default passwords that need to be changed
    - These can contain sensitive data
- Add additional security
    - Require addition logon
    - Add a 3rd party MFA

#### Removal of unnecessary software
- All software contains bugs
    - Some are security vulnerabilities
- Every application seems to have a completely different patching process
    - Challenging as there are many applications to manage
- Best practice is to delete applications you're no longer using
    - Reduce risk
    - Easy fix
