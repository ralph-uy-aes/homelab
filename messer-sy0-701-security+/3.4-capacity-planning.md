# 3.4 Capacity Planning

#### Capacity Planning
- How much capacity do you need?
    - Hard to say
- Match the supply to the demand
- Too much demand = Slowdowns and outages
- Too much supply = Wasted money
- There needs to be a balance
    - Right number of staff
    - Appropriate technology
    - Best infrastructure

#### People
- Some services require human intervention
    - Call center support lines
    - Technology services
- Too few employees
    - Recruit new staff
    - May be time consuming to add more staff
- Too many employees
    - Redeploy to other parts of organization
    - Downsize (layoffs)

#### Technology
- Pick a technology that can scale
- Web services
    - Use load balancers and multiple servers
- Database services
    - Split databases into multiple servers
    - Cluster multiple SQL servers
- Cloud services
    - Services on demand
    - Seems like unlimited resources if you pay enough money

#### Infrastructure
- The underlying framework
    - Applications servers
    - Network services
    - CPU, network, storage
- Physical devices
    - Purchase, configure, install
- Cloud-based devices
    - Easy deployment
    - Useful for unexpected capacity changes
