# 4.3 Penetration Testing

#### Penetration Testing
- PenTest
    - Simulated attack on own systems
    - A more in-depth vulnerability scan
    - Can we get access?
- Often a compliance mandate
- NIST - Technical Guide to Information Security Testing and Assessment

#### Rules of Engagement
- Formal rules defining purpose and scope of a pentest
- Also defines type of test and when it can be tested
- Rules can include
    - IP address ranges
    - Emergency contacts
    - Handling sensitive information
    - In-scope and out-of-scope devices or applications

#### Exploiting Vulnerabilities
- Try to break in
    - Can cause a DOS or data loss, so be careful
- May need to try many vulnerability types
    - Password brute-forcing
    - Social engineering
    - Databases injections
    - Buffer overflows
- You'll only be sure you're vulnerable if you can bypass security

#### The Process
- Initial Exploitation
    - Get into the network
- Lateral Movement
    - Go from one device to another
- Persistence
    - Make a backdoor, add a new account for yourself, or change passwords
- Pivot
    - Gain access to systems normally inaccessible
    - Use a vulnerable system as a proxy or relay

#### Responsible Disclosure Program
- Takes time to fix a vulnerability
    - Software changes, testing, deployment, etc
- Bug bounty programs
    - Rewards for finding vulnerabilities
- Controlled information release
    - Researcher reports vulnerability
    - Kept secret until a fix is found
    - Announced publicly thereafter
