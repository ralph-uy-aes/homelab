# 3.4 Backups

#### Backups
- Incredibly important
    - Fast and easy recovery for lost data and disasters
- Many different variables
    - Amount of data
    - Type of backup
    - Backup media
    - Storage location
    - Backup and recovery software
    - Day of the week for backups

#### Onsite vs Offsite
- On site
    - No internet link required
    - Less expensive, no need to rent extra space
    - Data is immediately available
- Off site
    - Transferred over the internet or WAN link
    - Data is available after a disaster
    - Restoration can be performed from anywhere
- Organizations often use both
    - More copies = more recovery options

#### Frequency
- How often to backup?
- May be different between systems
    - Some servers don't backup for months
    - Some backup every hour
- Backup sets
    - Four times a month, once a month, etc
- Requires significant planning

#### Encryption
- A history of data is on backup media
    - Some of this media may be offsite
- Attackers can easily break in and steal backups
- Protect backup of using encryption
    - Everything on the backup media is unreadable
    - Recovery key is required to restore data
- Especially useful for cloud

#### Snapshots
- Very common backup method for cloud and VMs
- Basically an exact copy of all configuration and data all in a click of a button
- Take a snapshot every 24 hours
- Easy to perform, easy to recover with

#### Recovery Testing
- Not enough to perform the backup
    - You have to have a good restoration and recovery pipeline
- Disaster recovery testing
    - Simulated disaster
    - Evaluate restoration
- Confirm the restoration
    - Test everything
- Perform periodic audits
    - Always have a good backup

#### Replication
- An ongoing, almost real-time backup
    - Keep data synchronized in multiple locations
- Data is available
    - There's always a real-time copy of data somewhere that can be used as a backup
- Data can be stored locally to all users
- Data is recoverable

#### Journaling
- Power goes out while writing data to storage
- Stored data is probably corrupted
- Recovery could be hard
    - Remove corrupted files, restore from backup
- Before writing to storage, make a journal entry
    - Write to journal first, and then write to storage
    - Prevents database corruption
- After data is written to storage, clear the journal and repeat
