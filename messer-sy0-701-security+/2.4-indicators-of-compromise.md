# 2.4 Indicators of Compromise

#### Indicators of Compromise
- An event that indicates an intrusion
    - High confidence that there is an intrusion
- Indicators
    - Unusual amount of network activity
    - Change to file hash values
    - Irregular international traffic
    - Changes to DNS data
    - Uncommon login patterns
    - Spikes of read requests to certain files

#### Account Lockout
- Credentials are not working, but it wasn't you
- Exceeded login attempts
- Account could have also been administratively disabled
    - Would be a larger concern if no one authorized disabled your account

#### Concurrent session usage
- Imagine being in Hawai'i and finding that your account is also logged in from France
- Can be difficult to track down
    - Multiple devices and desktops
    - Automated processes

#### Blocked Content
- An attacker wants to stay in as long as possible
    - Your system has been unlocked
    - Keep doors and windows open
- Blocked content
    - Viruses and malware tend to block auto-updates, links to security patches, third-party antivirus, and removal tools
    - This is so they can stay and be unevictable

#### Impossible Travel
- Authentication logs can be telling
- What if you log in and your location is in Hawai'i
    - But then a few seconds later you also login from France
- Easy to identify through log analysis and automation

#### Resource Consumption
- Every attacker's action has an equal and opposite reaction
    - Watch carefully for significant changes
- File transfers use bandwidth
    - Firewall logs show outgoing transfers
- Often first real notification of an issue

#### Resource Inaccessibility
- A down server could have been caused unintentionally or intentionally by an attacker
- Network disruption could be used to cover other processes an attacker is doing
- Encrypted data could indicate potential ransomware
- Brute force attacks cause lockouts

#### Out-of-cycle logging
- Out-of-cycle
    - Occurs at an unexpected time
- OS patch logs occurring outside of normal patch day
- Firewall log activity
    - Timestamps of every traffic flow
    - Protocols and applications used

#### Missing Logs
- Attackers could delete log information to hide their tracks
- Information is everywhere
    - Auth logs
    - File access logs
    - Firewall logs
    - Proxy logs
    - Server logs
- Logs may be incriminating
    - Missing logs are suspicious
    - Logs should be secured and monitored

#### Published/documented
- Exfiltrated private data are striking
- Such as blackmail or data leaks
    - This can be paired with ransomware by holding incriminating data hostage
- Happens quite often specially with movies, games, other media
- Knowing that your data has been pulled without authorization is a different story and could go unnoticed for a while up until the leak
