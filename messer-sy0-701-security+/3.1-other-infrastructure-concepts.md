# 3.1 Other Infrastructure Concepts

#### Attacks can happen anywhere
- Where should you store your data?
    - On-premise
    - Cloud-based
- Challenge of where is more secure
- Cloud-based
    - Centralized
    - Costs less
    - No dedicated hardware or data center to secure
    - Third-party handles everything
- On-premise
    - Client handles everything
    - Data center required
    - Staffing required
    - Infrastructure costs
- Attackers don't care where your data is, they will break in if they want to

#### On-premise Security
- Your security posture is what you make it, you have full-control
- Local IT Team may be able to manage security better
    - Local team can be expensive and difficult due to staff
    - Local team maintains uptime and availability, no need to phone cloud provider for support
- Security changes can take time
    - You handle new equipment, configurations, and other costs

#### Centralized vs Decentralized
- Most organizations are physically decentralized
    - Becomes more difficult to manage and secure with more systems
    - Security team needs to secure more moving parts
- Centralized approach
    - Correlated alerts
    - Consolidated log file analysis
    - Comprehensive system status and maintenance/patching
    - Not perfect as it leads to SPOF or potential performance issues


#### Virtualization
- Allows us to run many different OS on the same hardware
- Each application has its own OS, which can add overhead and complexity
    - Virtualization gets relatively 8expensive

#### Containerization
- Containers
    - Have everything you need to run an application, code and dependencies
    - Standard unit of software
- Every process believes it is isolated and cannot see or interact with other processes
- Container images are portable
    - Lightweight and shares the host kernel
    - Secure separation between applications
    - Can be built on virtual machines to share kernel with other OSes
- Instead of making two virtual machines to run 2 different Windows app, make one Windows VM with a Docker container

#### Internet of Things
- Found in many appliances, wearable technology, sensors, smart devices, automation
- IoT manufacturers tend to not secure their products too well
    - They tend to have weak defaults
    - IoT manufacturers aren't security professionals

#### SCADA/ICS
- Supervisory Control and Data Acquisition Systems
    - Think factory or industrial control systems (ICS)
- PC manages these systems which are connected via a network
    - Can control power, refineries, manufacturing equipment
    - Facilities, energy, logistics, etc...
- Control systems are distributed
    - Real-time information
    - System control
- As one could imagine, these need to be highly secure and requires extensive segmentation
    - No outside access, must be airtight
    - Think of the stuxnet worm

#### Real-Time Operating Systems (RTOS)
- Found in vehicles, military equipment, industrial equipment, etc
- Needs to have a deterministic processing schedule
- Pressing the brakes on your car is a safety feature, and RTOS ensures that the moment it's pressed, the braking process must be done
    - Determines processes in real-time
- Extremely sensitive to security issues
    - Non-trivial systems
    - Need to always be available
    - Difficult to know what type of security is in place

#### Embedded systems
- Hardware and software designed for one use and one use only, or to operate with a larger system acting as a function
    - Highly efficient at what they do
    - Can be optimized for size and/or cost
- Think: smartwatches, vapes, traffic light controllers, medical imaging systems

#### High Availability
- Big concern is to keep a system up and ready to use
- Redundancy can be used to ensure availability
- High Availability (HA)
    - Firewalls can have a backup firewall in case the main one fails
    - Always on, always available
- Can be hard to scale
    - More availability means more redundancy means more costs
- At some point, you'd have to decide whether the cost for downtime is worth the price for more redundancy
