# 4.5 Secure Protocols

#### Unencrypted Network Data
- Whatever is sent over the network isn't encrypted by default
- Protocols nowadays tend to be secure, but we still use unencrypted ones
    - Telnet, FTP, SMTP, IMAP
- In order to verify encryption, do a packet capture
    - If plaintext, it's unencrypted/unsecure

#### Protocol Selection
- Ideally, you should always use the secure protocol
- If you can't, just don't use the application or find a different one
- Insecure vs Secure Protocols
    - Telnet vs SSH
    - HTTP vs HTTPS
    - IMAP vs IMAPS
    - FTP vs SFTP
    - etc

#### Port Selection
- You can usually tell whether a protocol is secure or not based on the port
- Secure vs insecure protocols are ran on different ports
    - HTTP is on port 80
    - HTTPS is on port 443
- Port numbers do not guarantee security
    - Confirm the security features are enabled
    - Verify with a packet capture

#### Transport Method
- Applications cannot be reliable for encryption
    - Do it yourself
- 802.11 Wireless
    - Open AP: Not secure, no encryption
    - WPA3: All data is encrypted
- Virtual Private Network (VPN)
    - Create an encrypted tunnel
    - Traffic is encrypted and protected
    - Often requires third-party services and software
    - Might need to buy from VPN provider or buy your own VPN concentrator
