# 2.5 Segmentation and Access Control

#### Segmenting the network
- Physical logical or virtual segmentation
    - Devices, VLANs, virtual networks
- Performance
    - High-bandwidth applications
- Security
    - Users should not talk directly to database servers
    - Should go through applications which may have firewalls or control lists
- Compliance
    - Mandated segmentation (PCI Compliance)
    - Makes change control much easier

#### Access Control Lists (ACLs)
- Allow or disallow traffic
    - Groupings of categories
    - Source IP, Destination IP, Port number, Time of day, etc
- Restrict access to network devices
    - Limit by IP address or other identifier
    - Prevent regular user/non-admin access
- Be careful when configuring these
    - You can lock yourself out
- List the permissions
    - Bob can read files
    - Fred can access the network
    - James can access network 192.168.1.0/24 using tcp ports 80, 443, and 8088
- OSes use ACLs for file systems
    - Trustee and access rights

#### Application Allow/Deny list
- Any application can be dangerous
    - Vulnerabilities, trojan horses, malware, viruses
- Security policy can control app execution
    - Allow/deny list
- Allow list
    - Nothing runs unless it's approved
    - Very restrictive
    - White list
- Deny list
    - Nothing on the bad list can run
    - Anti-virus, anti-malware

#### Examples of allow and deny list
- Decisions are made in the OS
    - Often built-in to the OS management
- Application hash
    - Only allows applications with unique hashes
- Certificate
    - Allow digitally signed apps from certain publishers
- Path
    - Only applications from a certain folder
- Network Zone
    - Only run from certain network zones
