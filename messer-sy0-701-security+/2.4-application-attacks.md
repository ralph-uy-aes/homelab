# 2.4 Application Attacks

#### Injection Attacks
- Code injection
    - Adding your own information to a data stream
- Enabled because of lack of input validation
- So many different injectable data types

#### Buffer Overflow
- Overwriting a buffer of memory
    - Spills over into other memory areas
- Developers need to perform bounds checking
    - Attackers spend a lot of time looking for openings
- Not a simple exploit
    - Takes time to avoid crashing things
    - Takes time to make it do what you want
- A useful, repeatable, and consistent buffer overflow is really strong

#### Replay Attacks
- Useful information is transmitted over the network
- Needs access to raw network data
    - Network tap, ARP poisoning, Malware
- Gathered information can help the attacker
    - Replay the data to appear as someone else
- Not on-path attack

#### Privilege escalation
- Gain higher level access to system
    - Exploit a vuln
    - Might be a bug or design flaw
- Higher-level access means more capabilities
- High-priority vuln patches
    - You want to patch these quickly because any user can be an admin
- Horizontal privilege escalation
    - User A can access user B resources

#### Mitigating privilege escalation
- Patch quickly
- Update antivirus and block known vulns
- Data Execution Prevention
    - Only data in executable areas can run
- Address space layout randomization
- Prevent a buffer overrun at known memory addresses

#### Elevation of privilege vulnerability
- CVE-2023-29336
    - Win32k Elevation of privilege vulnerability
    - May 2023
- Win32k Kernel driver
    - Server 2008, 2008 R2, 2012, 2012 R2, 2016
    - Windows 10
- Attacker would gain System privileges
    - Which are the highest level access

#### Cross-site Request
- Common and legitimate
- Embedded youtube videos, instagram photos, etc...
- It looks like everything is coming from one server, but it actually uses many

#### Client and server
- Website pages consist of client-side codes
    - Many moving parts
- Client side
    - Renders page on screen
    - HTML, JS
- Server side
    - Performs requests from client
    - HTML, PHP
    - Transfer money from one account to another

#### Cross-site request forgery
- One-click attack, session riding
    - XSRF, CSRF
- Takes advantage of trust that a web application has for the user
    - Website trusts your browser
    - Requests are made without your consent or knowledge
    - Attacker posts a facebook status on your account
- Significant web application development oversight
    - Application should have anti-forgery techniques added
    - Usually a cryptographic token to prevent a forgery

#### Directory Traversal
- Directory traversal/ path traversal
- Read files from a web server that are outside of the website's file directory
- Users shouldn't be able to browse the Windows folder
- Web server software vulnerability
- Web application code vulnerability
