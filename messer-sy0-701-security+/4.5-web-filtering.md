# 4.5 Web Filtering

#### Content Filtering
- Control traffic based on web content
    - URL filter, website category filter
- Corporate control of inbound and outbound data
- Control for inappropriate content
- Protection against evil
    - Antivirus

#### URL Scanning
- Allow/Block list for Uniform Resource Locators
- Managed by category
    - Auction, Hacking, Malware, Travel, Recreation, etc.
- Can have limited control
    - URLs aren't the only way to surf
- Often integrated into an NGFW
    - Filters traffic based on category or URL

#### Agent Based
- Install client software on user's device
    - Usually managed from a central console
- Users can be located anywhere
    - Local agent makes filtering decisions
    - Always-on, always filtering
- Updates must be distributed to all agents

#### Proxies
- Sits between users and external network
- Receives user requests and sends the request on their behalf
- Can have a cache
- Provides extra security for URL filtering, access control, content scanning
- Explicit (visible) vs transparent (invisible)

#### Forward Proxy
- AKA internal proxy
- Commonly used to protect and control user access to the internet
- Outbound proxy

#### Block rules
- URL blocking
- Category blocking

#### Reputation
- Filter URLs based on perceived risk
    - Good reputation is allowed
    - Bad reputation is bocked
- Risk levels: trustworthy, low risk, medium risk, suspicious, high risk
- Automated reputation
    - Sites are scanned and assigned a reputation
- Manual reputation
    - If you don't agree with a rating, set it yourself


#### DNS Filtering
- Before connecting to a website, get the IP address
    - Perform a DNS lookup
- DNS is updated with real-time threat intelligence
    - Both commercial and public lists
- Harmful sites are not resolved
    - No IP address, no connection
- Works for any DNS lookup
    - Not just web filtering
