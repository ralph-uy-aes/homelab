# 3.2 Secure Infrastructures

#### Device Placement
- Every network is different based on its application and what an organization needs
- Firewalls
    - Help separate untrusted from trusted
    - Provides security checks
- Other services may add other devices such as honeypots, jump servers, load balancers, sensors, etc

#### Security Zones
- Zone-based security technologies are more flexible and secure than IP address ranges
- Areas in the network are associated with a zone
    - Examples are trusted, untrusted, internal, external, servers, databases, screened, etc
- This makes security policies easier to implement such as routing traffic from:
    - Trusted to Untrusted
    - Untrusted to Screened
    - Untrusted to Trusted

#### Attack Surface
- Know where attackers can get in and how
- Everything can be a vulnerability
    - Application code
    - Open ports
    - Authentication process
    - Human error
- Minimize surface
    - Audit code
    - Block ports on firewall
    - Monitor network traffic in real time

#### Connectivity
- Everything is connected and contributes to security
- Network cabling needs to be secured and physical drops need to be protected
- Application-level encryption is also an additional layer of protection
- Network-level encryption
    - IPsec tunnels, VPN connections
