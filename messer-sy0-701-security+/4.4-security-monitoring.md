# 4.4 Security Monitoring

#### Security Monitoring
- Attackers always be attacking
- Monitor all entry points
    - Check logins, remote access, physical locations, your available services
- React to security events
    - Account access, firewall rules, additional scanning
- Status dashboard
    - See status of all your systems at a glance

#### Monitoring Computing Resources
- Systems
    - Authentication - logins from strange places
    - Server monitoring - service activity, backups, software versions
- Applications
    - Availability - uptime and response times
    - Data transfers - increase or decrease in rates
    - Security notifications - from manufacturer
- Infrastructure
    - Remote access systems - employees, vendors, guests
    - Firewall and IPS - increase nad type of attack

#### Log Aggregation
- SIEM or SEM (Security Information and Event Manager)
    - Consolidates many different logs to a central database
    - Server, firewalls, VPN concentrators, SANs, cloud services
- Centralized reporting
- Correlation between diverse systems
    - Easier to compare and contrast information from multiple systems' logs

#### Scanning
- Constantly changing threat landscape
    - Hard to keep track of systems
- Actively check systems and devices
    - OS
    - Driver versions
    - Applications
    - Anomalies
- Gather raw information
    - Compile into a database

#### Reporting
- Analyze collected data
    - Create actionable reports
- Status information
    - What devices in the organization comply
    - What can be done for those that don't
- Determine best next steps
    - Create actions for improving after analysis
- Ad hoc information summaries
    - What can be done now to prepare for what might come?

#### Archiving
- Takes an average of about 9 months for a company to identify and contain a breach
    - IBM Security report 2022
- Access to data is critical
    - Archive over an extended period
- There may be mandates to archive
    - State or federal

#### Alerting
- Real-time notifications of suspicious security events
    - Large file transfers
    - Authentication errors
    - Odd location logins
- Actionable data
    - Keep people informed to have a quick response
- Notification methods
    - SMS/text
    - Email
    - Or only inform SOC through security console

#### Alert Response and Remediation
- Quarantine ASAP
    - Prevent lateral movement
    - Minimize attacker's scope
- Alerting is a balancing act
    - Don't always want alerts on false positives
    - But don't want to miss true positives
- Alerts should be accurate
    - Tune your security alerts properly
