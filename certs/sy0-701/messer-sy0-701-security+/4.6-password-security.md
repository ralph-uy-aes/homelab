# 4.6 Password Security

#### Password Complexity and Length
- Make your password strong
- Increase password entropy
    - No single words, no obvious passwords
    - Mix upper and lower case, number, and special characters
- At least 8 characters, better if longer

#### Password Age and Expiration
- Password age
    - How long since password was modified
- Password expiration
    - Works for intervals depending on admin
    - System remembers password history, new password needs to be different and unique
- Critical systems might need more frequent changes

#### Password Managers
- Important to use different passwords for each account
- Store all of your passwords in a single database
    - Encrypted, protected
    - Can include multifactor tokens
- Built-in to many OSes
- Also have some enterprise option

#### Passwordless Authentication
- Users tend to have weak passwords, so just don't use one
- Authenticate without a password
    - Facial recognition, security key, etc

#### Just-in-time Permissions
- Allows temporary admin rights
- Limited time, automatic timeout
    - Least privilege
- Request access from a central clearinghouse
- Password vaulting
    - Vault controls who gets access
- Accounts are temporary
    - Creates a time-limited account
    - Admin receives ephemeral credentials
    - Primary passwords are never released
    - Credentials are used for one session then deleted
