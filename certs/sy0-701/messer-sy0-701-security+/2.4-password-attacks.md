# 2.4 Password Attacks

#### Plaintext/ Unencrypted passwords
- Some apps store passwords in the clear
    - Meaning no encryption
- Do not store passwords as plaintext
    - Anyone with access to a password file or DB has every credential
- What if you find this?
    - Contact app devs

#### hashing a password
- Hashes represent data as a fixed-length string of text
    - Message digest or fingerprint
- Will not have a collision (hopefully)
    - Different inputs = different outputs
- One-way trip
    - Hashes cannot be used to get passwords
    - Fingerprints cannot be used to reconstruct humans

#### Password files
- May have unencrypted usernames, but encrypted passwords

#### Spraying attacks
- Trying to log in with an incorrect password
- Attack an account with the top three or more passwords
    - If these don't work, move to the next account
    - No lockouts, no alarms, no alerts

#### Brute force
- Try every possible password combination until hash is matched
- Might take some time
- A strong hashing algorithm slows things down
- Getting a password hash and then trying to find a password that matches the hash is easier as some logins have a lockout or login limit
