# 4.5 Email Security

#### Email Security Challenges
- Protocols used to transfer emails include few security checks
    - Very easy to spoof an email
- Spoofing happens all the time
    - Hard to tell who sent you what
- Reputable sender will configure email validation
    - Publicly available on the sender's DNS server

#### Mail Gateway
- The Gatekeeper
    - Evaluates source of inbound email messages
    - Blocks at the gateway before it reaches the user
    - On-site or cloud-based

#### Sender Policy Framework (SPF)
- SPF protocol
    - Defines which authorized email servers can send mail on your behalf
- List of authorized mail servers are added to a DNS TXT record
    - Receiving mail servers perform a check to see if incoming mail really did come from authorized host

#### Domain Keys Identified Mail (DKIM)
- Mail server that automatically digitally signs all outgoing mail
- Public key is in the DKIM TXT record
- Signature is validated by the receiving mail servers
    - Not usually seen by end user

#### Domain-based Message Authentication, Reporting, and Conformance (DMARC)
- Extension of SPF and DKIM
- Domain owner decides what receiving email servers should do with emails not validating using SPF and DKIM
    - Policy is written into DNS TXT record
- Compliance reports are sent to email admin
    - Domain owner can see how many emails are received
