# 3.2 Network Appliances

#### Jump Server
- A server you can use to manage your internal devices from outside of your network
- Jump servers must be highly secured, hardened, and monitored
- You can SSH/Tunnel/VPN into the jump server
    - RDP, SSH, or jump from there to control internal devices
- If your jump server gets compromised, everything else gets compromised

#### Proxies
- Something that sits between users and external networks
- Relays traffic to the internet or to external networks that it picks up from other devices
- Might seem useless, but are actually great for caching information, access control, URL filtering, content scanning
    - If multiple devices connect to the same proxy, all of those devices will have some sort of collective cache
- Explicit proxies
    - Applications need to know how to use the proxy
- Transparent proxies
    - Users don't even know they're communicating via a proxy

#### Application Proxies
- One of the simplest proxies is the NAT (network-level)
- Most people think of application-level proxies such as ones for protocols like HTTP, HTTPS, FTP

#### Forward Proxy
- An internal proxy is used to be a middleman between user and the internet
- Any traffic routed from the user is sent to the internet
- Any traffic from the internet is examined before being relayed to the user

#### Reverse Proxy
- Middleman between your web server and the internet
- Examines inbound traffic from the internet and then relays to web server
- If a user requests for some data, it is cached in the proxy for other users doing subsequent requests
- Think of DNS resolvers

#### Open Proxy
- Significant security concern
- Managed by some third-party as opposed to being an internal proxy
    - You don't know what they're doing with your traffic
- Can be used to circumvent security controls

#### Load Balancers
- Distributes loads between multiple servers
    - Invisible to end-users
- Good for large-scale or enterprise use
    - Web server farms, database farms, or anything that has frequent and many requests
- Also provides fault tolerance
    - Server outages have no effect
    - If one goes down, load balancer just reroutes traffic and splits it to other servers
    - Convergence is very fast

#### Active/active load balancing
- Active/active means every server is active and being used
- Can provide TCP offloading
    - No need for every server to make a TCP connection
    - Each server connects to the load balancer which only needs one TCP connection to the internet
- Can provide SSL offloading
    - Servers don't need to encrypt/decrypt individually
    - Load balancers handled it by encrypting data from servers and decrypting data from the internet
- Caching
- Prioritization
    - QoS
- Content Switching
    - Application Centric Balancing

#### Active/passive load balancing
- Some servers are on standby
- If an active server fails, passive server takes its place

#### Sensors and Collectors
- Aggregate information from network devices
    - Sensors get data, Collectors interpret that data
    - Already integrated into switches, routers, servers, firewalls, etc
- Sensors
    - IPS, Firewall logs, auth logs, web server access logs, database transaction logs, email logs
- Collectors
    - Proprietary consoles (IPS, firewall), SIEM Consoles, syslog servers
    - Many SIEMs include correlation engine to compare diverse sensor data
