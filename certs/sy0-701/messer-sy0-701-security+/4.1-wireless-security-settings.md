# 4.1 Wireless Security Settings

#### Securing a Wireless Network
- A concern is that everything is sent over the air
    - Anyone can intercept and manipulate frames or packets
- To prevent this, authenticate users before granting access to a network
    - Only allow trusted users
    - Username, password, MFA
- Ensure all communication is confidential
    - Encrypt all data being sent wirelessly
- Verify the integrity of messages
    - Do a Message Integrity Check (MIC)
    - Data received should be identical to data sent

#### WPA2 PSK Problem
- Wi-Fi Protected Access 2 (WPA2) uses a pre-shared key (PSK) hash
    - A PSK is basically your Wi-Fi password
    - If an attacker listens to the WPA2 four-way handshake and captures the PSK hash, they can brute-force it to get the PSK
- Today breaking a hash is extremely easy with faster GPUs and cloud-based password cracking
- Once you have the PSK, you have everyone's wireless key
    - No forward secrecy

#### WPA3 and GCMP
- WPA3 was introduced in 2018 alongside Galois/Counter Mode Protocol (GCMP)
- Stronger encryption than WPA2
- GCMP security services provides
    - Data confidentiality with AES
    - Message integrity check (MIC) with Galois Message Authentication Code (GMAC)

#### SAE
- WPA3 changes the PSK authentication process
    - Has mutual authentication
    - Creates a shared session key without sending that key across the network
    - No more four-way handshakes, meaning no more hashes sent over the network to brute-force
- Simultaneous Authentication of Equals (SAE)
    - A Diffie-Hellman derived key exchange with an authentication component
    - Everyone uses a different session key, even with the same PSK
    - IEEE standard calls it the dragonfly handshake

#### Wireless Authentication Methods
- Gain access to a wireless network by either credential
    - PSK or the Wi-Fi's password
    - Centralized authentication (802.1X), uses a RADIUS/LDAP/TACACS server

#### Wireless Security Modes
- Configure authentication on your WAP/router
- Open-System
    - No password, anyone can connect
    - Bad
- WPA3-Personal/WPA3-PSK
    - Typical for home networks, it's just network SSID + password
    - Everyone uses the same 256-bit key
- WPA3-Enterprise/WPA3-802.1X
    - Authenticates users individually with an authentication server

#### AAA Framework
- Identification
    - Who are you?
- Authentication
    - Prove it (username/password)
- Authorization
    - What do you have access to
- Accounting
    - We're watching you

#### RADIUS
- One of the most common AAA protocols
- Remote Authentication Dial-In User Service
- Centralize authentication for users
    - If you provide credentials, it is compared to a AAA server's database and if it matches, you may connect
    - Routers, switches, firewalls
    - Server authentication
    - Remote VPN access
    - 802.1X network access
- RADIUS are available on almost any server OS

#### IEEE 802.1X
- AKA Network Access Control or NAC
- The pop-up that we see asking for our credentials before being able to connect
- Used in conjunction with access databases like RADIUS, LDAP, or TACACS+

#### EAP
- Extensible Authentication Protocol (EAP)
    - An authentication framework
- Many different ways to authenticated based on the RFC standards
    - Manufacturers can build their own EAP methods
- EAP integrates with 802.1X
    - Prevents unauthorized access ot network

#### IEEE 802.1X and EAP
- Supplicant - the client
- Authenticator - device that provides access
- Authentication Server - validates client credentials
