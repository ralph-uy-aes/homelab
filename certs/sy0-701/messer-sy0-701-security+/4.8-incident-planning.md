# 4,8 Incident Planning

#### Exercising
- Test yourself before an actual event periodically
    - Create a test environment, not production
    - Make rules and scenarios
    - Limit time to respond
    - Evaluate response and improve

#### Tabletop Exercises
- Full-scale drills are 
    - Time-consuming, costly, impractical
- Logistics can be determined through analysis
- Get key players together for a tabletop exercise
    - Talk through a simulated disaster

#### Simulation
- Test with a simulated event
    - Phishing attack, password requests, data breaches
- Going Phishing
    - Security team attempts a simulated phishing attack
    - Biters get retrained
- Test internal security
    - Did the phishing get past the filter?
    - How did your other systems respond?
- Test the users
    - Who clicked?

#### Root Cause Analysis
- Determine the ultimate cause of the incident
    - How did they even get a foothold?
- Create a set of conclusions regarding the incident
- Don't get tunnel vision
    - Attackers typically have more than one entry
    - Meaning, there is more than one root cause
- Users make mistakes
    - Response to mistakes determines the outcome
    - How do we prevent any future attacks

#### Threat Hunting
- Finding the vulnerability before attackers do
- Intelligence data is reactive
    - You can't see the attack until it happens
- Speed up the reaction time
    - Use technology to fight
