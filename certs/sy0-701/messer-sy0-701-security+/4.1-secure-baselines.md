# 4.1 Secure Baselines

#### Secure Baselines
- Security of an application environment should be well-defined
    - All application instances must follow this baseline
    - Firewall settings, patch settings, OS file versions, etc must work towards best practices
    - May require constant updates
- Secure baselines must be met by application whenever it's deployed
- Do integrity measurements to check for the secure baseline
    - Should be performed often
    - Check against well-documented baselines
    - Failure requires immediate correction

#### Establish Baselines
- Create a series of baselines
    - Foundational security policies
- Or look for third-party baselines
    - Manufacturers often create these baselines, just contact them
- Many OS have extensive options
    - Over 3,000 group policy settings in Windows 10
    - Only a subset is actually used in security 

#### Deploy Baselines
- We now have established detailed security baselines
    - How do we put those baselines into action
- Deploy baselines
    - Usually managed through administered console
- May require multiple deployment mechanisms
    - Active Directory group policy, MDM, etc.
- Automation is key
    - Deploy to hundreds or thousands of devices

#### Maintain baselines
- Many of these are best practices, rarely change
- Other baselines may require ongoing updates
    - New vulnerabilities, new updates, new OS
- Test and measure to avoid conflicts
    - Baselines can contradict each other
