# 2.3 Misconfiguration Vulnerabilities

#### Open Permissions
- Simply leaving the door open
    - Hackers will find it 
    - Leaving your data in a cloud storage unsecured
- Increasingly common
    - Statistical chance of finding an open permission
- June 2017 - 14 million Verizon records exposed
    - An Amazon S3 data repository was left open
    - A researcher found all the exposed data before anyone else

#### Unsecured Admin Accounts
- Root/Sudo/Admin account
- Can be a misconfiguration
    - Easy to hack passwords
- Disable direct login to the root account
    - Use the su or sudo option
- Protect accounts with root or administrator access
    - Shouldn't be alot of these

#### Insecure Protocols
- Some protocols aren't encrypted
    - All traffic sent in plaintext
    - Telnet, HTTP, FTP, SMTP, IMAP
- Verify with packet capture
- Use the encrypted versions
    - HTTPS, SSH, SFTP

#### Default Settings
- Every application and network device has a default login
    - Not all have been changed since setup
- Mirai botnet
    - Takes advantage of default configurations
    - Takes over IoT devices
    - 60+ default configurations
    - Cameras, routers, doorbells, garage doors, etc...
- Mirai is released as open-source, meaning anyone can use it

#### Open Ports and Services
- Services will open ports
- Ports must be managed with a firewall
    - Manage traffic flows
    - Allow or deny based on port number or application
- Firewall rulesets can be complex and can make mistakes
    - Always test and audit
    - Double and triple check the rulesets
