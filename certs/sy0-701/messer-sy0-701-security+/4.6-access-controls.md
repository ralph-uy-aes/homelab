# 4.6 Access Controls

#### Access Control
- Authorization
    - Process of ensuring only authorized rights are exercised
    - Policy enforcement and definition
    - Process of determining rights
- Users receive rights based on AC models
    - Different business needs or mission requirements

#### Least Privilege
- Security best practice
- Assign rights and permissions for user with only what they need to do their job
- All user accounts must be limited
    - Applications should run with minimal privilege
- Don't allow users to run with admin privilege
    - Limit the scope of malicious behavior

#### Mandatory Access Control (MAC)
- OS limits operation on an object
    - Based on security clearance labels
- Every object gets a label
    - Confidential, secret, top secret, etc
- Labeling of objects uses predefined rules
    - Administrator decides who gets access to what security level
    - Cannot change settings

#### Discretionary Access Control (DAC)
- Used in most OS
- Think Google suite
    - Owner controls who has access, modifiable at any time
- Very flexible AC

#### Role-based Access Control (RBAC)
- You have a role in your organization
    - Manager, director, team lead, project manager
- Admins provide access based on role of user
    - Rights are gained implicitly instead of explicitly
- In Windows, use Groups to provide RBAC
    - Based on your role in a group or your group's role

#### Rule-Based Access Control
- Generic term for following rules
    - Conditions other than who you are
- Access is determined through system-enforced rules
    - Sys admins, not users
- Rule is associated with the object
    - System checks ACLs for that object
- Rule examples
    - Lab network access only between 9AM to 5PM
    - Only Chrome browsers may complete the web form

#### Attribute-Based Access Control (ABAC)
- Many different criteria
- Next generation auth model
    - Context-aware
- Combines and evaluates a lot of parameters
    - Resource information, IP address, time, action, relationship to data
 
#### Time-of-day Restriction
- Almost all security devices have a ToD option
- Difficult to implement
    - Especially in a 24-hour environment, international organization
    - Can be more granular, including time zones
- Usually not standalone, used with other AC restrictions
