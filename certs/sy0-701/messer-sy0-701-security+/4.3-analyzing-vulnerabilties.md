# 4.3 Analyzing Vulnerabilities

#### False Information
- False positives
    - Vulnerability is found, but it doesn't really exist
    - Different than a low-severity vulnerability
- False Negative
    - Vulnerability exists, but you didn't detect it
    - Really bad
- Update to the latest signatures
    - Updated scans = more accurate results + new vulnerabilities
    - Work with vulnerability detection manufacturer

#### Prioritizing Vulnerabilities
- Not every vulnerability shares the same priority
    - Some may not be significant
    - Others may be critical
- May be difficult to determine
    - Research has already probably been done
- Refer to public disclosures and vulnerability databases

#### CVSS
- National Vulnerability Database is in nvd.nist.gov
- Common Vulnerability Scoring System
    - 0 to 10, least to most critical
    - Scoring standards change over time
    - Different versions CVSS 2.0 vs CVSS 3.x
- Industry Collaboration
    - Enhanced feed sharing and collaboration

#### CVE
- Common Vulnerabilities and Exposures, cve.mitre.org
- Vulnerabilities can be cross-referenced online
- Some vulnerabilities cannot be definitively identified
    - You'll have to check manually
    - Or research by yourself

#### Vulnerability Classification
- Scanner looks for everything
- Application scans
    - Desktops, Mobile apps
- Web application scans
    - Software on a web server
- Network scans
    - Misconfigured firewalls, open ports, vulnerable devices

#### Exposure factor
- Loss of value or business activity if the vulnerability is exploited
    - Usually expressed as a percentage
- Small DDoS may limit access to a service
    - ~50% exposure factor
- Buffer overflow may completely disable a service
    - 100% exposure factor
- Consideration when prioritizing

#### Environmental Variables
- What kind of environment is the vulnerability in?
    - Server, cloud, test lab
- Prioritization and patching frequency
    - Which environment to prioritize on?
- Every environment is different
    - User base
    - Revenue
    - Exploit potential

#### Industry/Organizational Impact
- Some exploits have significant consequences
- Tallahassee Memorial Healthcare, Feb 2023
    - Ransomware - closed for 2 weeks
    - Emergency cases cannot be responded to, surgeries cancelled
- Power Utilities - March 2019
    - DDoS attacks from unpatched known vulnerability

#### Risk Tolerance
- Which patch is prioritized?
    - Cannot download all simultaneously
- Impractical to remove all risk
    - Too hard
    - Instead, how much risk can you tolerate
- Timing of security patches
    - While you're testing, you're also vulnerable
- Middle ground must be found
