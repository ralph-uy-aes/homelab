# 2.4 DNS Attacks

#### DNS Poisoning
- Modify the DNS server
    - Requires some crafty hacking
- Modify the client host file
    - Host file takes precedent over DNS queries
    - Change IP address of legitimate site to the IP of a potentially malicious site
- Send a fake response to a valid DNS request
    - Requires a redirection of the original request or resulting response
    - Real-time redirection
    - On-path attack

#### Domain Hijacking
- Get access to domain registration, and you have control where the traffic flows
    - You don't need to touch actual servers
    - Determines DNS names and DNS IPs
- Many ways to get into the account
    - Brute force
    - social engineering
    - Gain access to email address that manages accounts
    - Usual things
- October 22, 2016 Brazilian Bank
    - 36 domain name registrations were changed
    - Under hacker control for 6 hours
    - Attackers became the bank
    - 5 Million customers, 27 Billion dollars in assets affected

#### URL Hijacking
- Make money from mistakes
- Sell badly spelled domain that's similar to a legitimate domain
- Redirect to a competitor or a phishing site
    - Or infect with a drive-by download

#### Types of URL Hijacking
- Typosquatting/Brandjacking
    - Take advantage of poor spelling
- Intentional misspelling
    - google.com vs goog1e.com
- Different phrase
    - googles.com
- Different TLD
    - google.org (which might actually be legitimate)
- Big companies usually buy out these hijacked URLs to keep their customer base, price is worth the traffic
