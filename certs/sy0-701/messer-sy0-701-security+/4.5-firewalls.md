# 4.5 Firewalls

#### Network-based Firewalls
- Traditional - filter traffic by port number
- NGFW - filter traffic by application
- Encrypt traffic, VPN between sites
- Most firewalls can be layer 3 devices
    - Often sits on ingres/egress of a network
    - Network Address Translation (NAT)
    - Dynamic Routing

#### NGFW
- OSI layer 7 firewall
- Can also be called
    - Application layer gateway
    - Stateful multilayer inspection
    - Deep packet inspection
- Requires some advanced decodes
    - Analyze, categorize, decide based on captures packets

#### Ports and Protocols
- Traditional firewalls could only make forwarding decisions based on protocol and port number
- Based on destination protocol and port
    - Web server: tcp/80 and tcp/443
    - SSH server: tcp/22
    - Microsoft RDP: tcp/3389
    - DNS query: udp/53
    - NTP: udp/123

#### Firewall Rules
- A logical path
    - Top to bottom
- Can be general or specific
    - Specific rules are at the top
- Implicit deny
    - Most firewalls include a deny at the bottom
- Access Control Lists (ACL)
    - Allow or disallow traffic
    - Groupings of categories
        - Source/Dest IP
        - Port number
        - Time of day
        - Application
        - Etc.

#### Screened Subnet
- Additional layer of security between you and the internet
- Public access to public resources
- Private data stays inaccessible

#### IPS Rules
- Intrusion Prevention System
    - Usually included with NGFW
- Different ways to find malicious traffic
- Signature-based
    - Look for a perfect match
- Anomaly-based
    - Build a baseline of what's normal
    - Unusual traffic patterns are flagged
- You determine what happens when unwanted traffic appears
    - Block, allow, send alert, etc
- Thousands of rules or more
- Rules can be customized by group
- This can take time to find the right balance
