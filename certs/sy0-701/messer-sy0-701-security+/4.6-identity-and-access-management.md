# 4.6 Identity and Access Management

#### Identity and Access Management (IAM)
- We provide the right permissions to the right people at the right time with IAM
    - It prevents unauthorized access
- Identity Lifecycle Management
    - Every entity gets a digital identity
- Access Control
    - Entities only get whatever access they need
    - Need basis
- Authentication and Authorization
    - Entities must prove they are who they claim to be
- Identity governance
    - Entities' resource access are tracked

#### Provisioning/de-provisioning User Accounts
- Adding and removing a user must have a formal process
- Hiring, transfers, promotions, job separation, etc
- Account details must be included
    - Names, attributes, group permissions, other permissions
- Important for the IAM process
    - Checkpoint limits access
    - Nobody gets admin access

#### Permission Assignments
- Entities get limited permissions
    - Least privilege
    - Typically in groups
- Entity can only access their own storage and files
    - No OS access
        - Prevents third-party OS access
        - Prevents privileged changes to OS
    - No access to other entities' storage and files

#### Identity Proofing
- There should be a formal process in verifying who a user is upon creation
- Resolution
    - Who the system think you are
- Validation
    - Gathering information from the user to prove your identity
- Verification/Attestation
    - Passport, In-person meeting, etc
    - Automated verification is also an option

#### Single Sign-on (SSO)
- Provide credentials one time
    - Get access to all available or assigned resources
    - No additional authentication required
- Usually limited by time
    - Authenticate again after timer expires
- Underlying authentication infrastructure must support SSO
    - Not always an option

#### Lightweight Directory Access Protocol (LDAP)
- Protocol for reading and writing directories over an IP network
    - Organized set of records, like a phone directory
- X.500 spec written by the International Telecommunications Union (ITU)
- DAP ran on the OSI protocol stack, LDAP is lightweight
- It is the protocol used to query and update an X.500 directory
    - Used in Windows AD, Apple OpenDirectory, Novell eDirectory, etc

#### X.500 Distinguished Names
- `attribute=value` pairs
- Most specific attribute is listed first
    - CN = Common Name
    - OU = Organizational Unit
    - O = Organization
    - L = Locality
    - ST = State
    - C = Country
    - DC = Domain Component
- Structure within a database
    - Can be used to build a hierarchical tree
- Container objects
    - Country, organization, organizational units
- Leaf objects
    - User, computers, printers, files

#### Security Assertion Markup Language (SAML)
- Open standard for authentication and authorization
    - Can authenticate through third-party to gain access
    - One standard kinda does it all
- Not originally designed for mobile apps
    - Biggest roadblock for SAML
- Process
    - User attempts to access application URL, but they're not logged in
    - Application server redirects them to Authorization Server with encrypted SAML request
    - User logs in to Auth server
    - Authentication successful, Auth server creates a SAML token
    - User presents SAML token to App server and gets access

#### OAuth
- Authorization framework
    - Determines what resources a user will be able to access
- Created by big conglomerates, lots of industry backing
- Not an authentication protocol
    - OpenID Connect handles the SSO
    - OAuth provides the authorization between applications

#### Federation
- Provide network access to others without local auth database
    - Think of the "Log In with Facebook or Apple or whatever" button
- Third-parties can establish a federated network
    - Authenticate and authorize between two separate organizations
    - Obviously this requires a formal trust relationship between the two parties before using existing login credentials

#### Interoperability
- Many different ways to communicate with an authentication server
- Must ensure that new systems are compatible 
    - Typically "what is compatible with existing systems"
- Dependent on the environment, part of a much larger IAM strategy
