# 4.7 Scripting and Automation

#### Scripting and Automation
- When situations occur, you don't need to handle them, just use an automated script
- Faster than you, effortless, accurate, and automatic

#### Automation Benefits
- Saves time
    - No typing
    - Rerunnable
- Enforce Baselines
    - Missing an important security patch?
    - Automatically install when identified
- Standard infrastructure configurations
    - Use a script to build default device configs
- Secure Scaling
    - Quickly scale up and down
    - Automation ensures proper security also scales
- Employee Retention
    - Automate the boring stuff, ease the workload
    - Employees work is rewarding, not mundane and repetitive
- Reaction time
    - Much faster than you
    - Events are handled immediately
- Workforce multiplier
    - Runs 24/7

#### Cases for Automation
- User and resource provisioning
    - Employee on-boarding and off-boarding
    - Assign access to specific resources
- Guard rails
    - Set of automated validations
    - Limits behaviors and responses
    - Check constantly to ensure proper implementation
    - Reduce errors
- Security Groups
    - Add or remove users automatically from groups
    - Constant audits without human intervention
- Ticket Creation
    - Automatically identify issues
    - Script email submissions into a ticket
- Escalation
    - Correct issues before involving a human
    - If not easily resolvable, call a tech
- Controlling services and access
    - AUtomatically enable and disable services
- Continuous integration adn testing
    - Constant updates, tests, and deployments
- Integrations and APIs
    - Interact with third-party devices and services

#### Scripting Concerns
- Complexity
    - Lots of moving parts
- Cost
    - Takes money to create a script and implement automation
- SPOF
    - What happens if the script stops working
- Technical debt
    - Patching problems may push the issue down the road
- Ongoing supportability
    - Script must have changes and update
