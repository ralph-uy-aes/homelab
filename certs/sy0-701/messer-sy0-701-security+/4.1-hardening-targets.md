# 4.1 Hardening Targets

#### Hardening Targets
- No system is secure with default configs
- Many manufacturers provide hardening guides to help protect products out of the box
- There are also some general-purpose guides available online

#### Mobile Devices
- Always-connected mobile technologies
    - Phones, tablets
    - Hardening checklists are available from manufacturers
- Updates are critical
    - Bug fixes and security patches
    - Prevent known vulnerabilities
- Segmentation can protect data
    - Company and user data are separated
    - Access to one segment, but not another
- Control with an MDM
    - Mobile Device Manager

#### Workstations
- User desktops and laptops
    - Windows, MacOS, Linux
- Constant monitoring and updates
    - OS, apps, firmware, drivers
- Automate the monthly patches
    - Likely has an existing process
- Connect to a policy management system
    - Active Directory group policy
- Remove unnecessary software limits the threats
    - Reduces attack surface

#### Network Infrastructure Devices
- Switches, routers, etc
    - You never see them, but they're always there
- Purpose-built devices
    - Embedded OS, limited OS access
- Configure authentication
    - Don't use the defaults
- Check with manufacturer
    - Security updates
    - Not usually updated frequently, so an update is usually big or major


#### Cloud Infrastructure
- Secure cloud management workstation
    - Keys to the kingdom (all your data)
- Least privilege
    - All services, network settings, application rights and permissions
- Configure Endpoint Detection and Response (EDR)
    - All devices accessing the cloud should be secure
- Always have backups
    - Cloud to Cloud (C2C)

#### Servers
- Many and varied
    - Windows, Linux, etc
- Updates
    - OS updates/service packs, security patches
- User accounts
    - Minimum password lengths and complexity
    - Account limitations
- Network Access and security
    - Limit network access
- Monitor and secure
    - Anti-virus

#### SCADA/ICS
- Supervisory Control and Data Acquisition System
    - Large scale, multi-site Industrial Control Systems (ICS)
- PC manages equipment
    - Power generation, refining, manufacturing equipment
    - Facilities, industrial, energy, logistics
- Distributed control systems
    - Real-time information
    - System control
- Devices tend to be very well secure, often have their own isolated network
    - Extensive segmentation, only intranet

#### Embedded Systems
- Hardware and software designed for a specific function
- Limited access to OS
    - Can be difficult to upgrade
- Correct vulnerabilities as soon as possible
    - Security is often limited for these devices
- Segment and firewall these embedded systems ideally

#### RTOS (Real-Time Operating Systems)
- An OS with deterministic processing schedule
- Isolate the system
    - Prevent access from other areas
- Run with minimum services
    - Prevent potential for exploit
- Use secure communication
    - Protect with host-based firewall

#### IoT Devices
- IoT manufacturers often have weak security defaults
- Often have weak security in general
- Deploy updates quickly
- Segment IoT devices on their own VLAN
