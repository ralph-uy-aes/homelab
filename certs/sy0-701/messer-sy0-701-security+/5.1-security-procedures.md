# 5.1 Security Procedures

#### Change Management
- Formal process for managing change
    - Avoid downtime, confusion, mistakes
- Nothing changes without the process
    - Determine the scope of the change
    - Analyze risk associated with the change
    - Create a plan
    - Get end-user approval
    - Present proposal to change control board
    - Have a backout plan for the change
    - Document changes

#### Onboarding
- Bringing a new person into the organization
- IT agreements need to be signed
    - May be a part of the employee handbook or a separate AUP
- Create accounts
- Provide required IT hardware

#### Offboarding
- Process should be pre-planned
- What happens to user's assets upon leaving organization
    - Data
    - Hardware
- Accounts are usually deactivated, not deleted

#### Playbooks
- Conditional steps to follow in case of a particular event
    - Data breach playbook, ransomware playbook, etc
- Usually a lot of playbooks
    - Step-by-step procedures
- Often integrated with a SOAR platform
    - Security Orchestration, Automation, and Response
    - Integrate third-party tools and data sources
    - Make security teams more effective

#### Monitoring and Revision
- IT security is constantly changing
    - Processes and procedures also must change
- Update to security posture
    - Tighter change control, additional playbooks
- Change to an individual procedure
    - Update playbooks, include additional checks
- New security concerns
    - Protect against emerging threats

#### Governance Structures
- Boards
    - Panel of specialists
    - Set the tasks or requirements for committees
- Committees
    - Subject-matter experts
    - Considers input and feedback from a board
    - Determines next steps for topic at hand
    - Presents results to the board
- Government entities
    - Different kind of machine
    - Legal concerns, political issues, administrative requirements
    - Often open to the public
- Centralized/decentralized
    - Source of processes and procedures
    - Centralized - one location with a group of decision makers
    - Decentralized - spreads decision-making processes to other individuals/locations
