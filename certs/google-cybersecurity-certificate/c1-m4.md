# Course 1 – Module 4: Tools for Protecting Business Operations

Entry-level analysts use a combination of industry-standard tools and playbooks to monitor, investigate, and respond to threats.

## Key Tools

| Tool | Purpose |
|------|---------|
| **SIEM** | Collects and analyzes log data; provides dashboards and alerts for threats. Hosted on‑premise or cloud. |
| **Network protocol analyzer (packet sniffer)** | Captures and inspects data traffic within a network. |
| **Playbooks** | Manuals detailing step‑by‑step procedures for incident response and other operational tasks. |

### Playbook Examples
- **Chain of custody** – Documents evidence possession and control throughout the incident lifecycle.
- **Protecting and preserving evidence** – Guides handling of fragile/volatile digital evidence using the **order of volatility** (prioritizes data most likely to be lost).

---

## Additional Tools and Concepts

- **Programming** – Automates repetitive tasks. Common languages:
  - **Python** – for automation.
  - **SQL** – for querying databases.
- **Operating systems** – Linux, macOS, Windows. Linux is open‑source and uses a command‑line interface.
- **Web vulnerability** – Flaws in web apps that attackers exploit. Refer to **OWASP Top 10** for current risks.
- **Antivirus software (anti‑malware)** – Prevents, detects, and removes malware.
- **Intrusion Detection System (IDS)** – Monitors system activity and alerts on intrusions by analyzing network packets.
- **Encryption** – Converts plaintext into ciphertext to ensure confidentiality. (Encoding is different—it uses public algorithms for data sharing.)
- **Penetration testing (pen testing)** – Simulated attacks to identify vulnerabilities in systems and processes.

*The more tools you know, the more valuable you are to an organization.*

# Creating a Cybersecurity Portfolio

A portfolio demonstrates your skills, knowledge, and passion to potential employers—it goes deeper than a resume.

## Portfolio Hosting Options

| Option | Description |
|--------|-------------|
| **Documents folder** | Local folder on your hard drive; organize with subfolders (resume, education, projects, etc.). |
| **Google Drive / Dropbox** | Cloud storage with sharing features; updates sync automatically. |
| **Google Sites** | Website builder with responsive layouts; publish a unique URL for your resume. |
| **Git repository** | Use GitLab, Bitbucket, or GitHub to store files and showcase work using Markdown. |

## Portfolio Projects (from the program)

- Professional statement
- Security audit
- Network structure analysis
- Linux file permissions
- SQL queries with filters
- Vulnerability identification
- Incident handler’s journal
- Python text file parsing
- Resume creation

*Keep your site private until finalized and never include private, copyrighted, or proprietary documents.*

# Module 4 Glossary

- **Antivirus software** – Detects and eliminates malware.
- **Database** – Organized collection of data.
- **Data point** – A specific piece of information.
- **IDS** – Monitors activity and alerts on intrusions.
- **Linux** – Open‑source operating system.
- **Log** – Record of events within an organization's systems.
- **Network protocol analyzer (packet sniffer)** – Captures and analyzes network traffic.
- **Order of volatility** – Sequence for preserving data from most to least volatile.
- **Programming** – Creating instructions for computers to execute tasks.
- **Protecting and preserving evidence** – Proper handling of fragile/volatile evidence.
- **SIEM** – Collects and analyzes log data for monitoring and alerts.
- **SQL** – Language for creating, interacting with, and querying databases.
